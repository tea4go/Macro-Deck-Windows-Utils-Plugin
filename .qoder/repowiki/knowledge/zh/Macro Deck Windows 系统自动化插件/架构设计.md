- **入口与注册**：`Main.cs` 继承 `MacroDeckPlugin`，在 `Enable()` 中实例化并注册所有 `PluginAction` 子类。
- **动作实现层**：`Actions/` 目录下各动作类（如 `StartApplicationAction`）继承 `PluginAction`，通过 `Trigger` 执行逻辑，`GetActionConfigControl` 返回配置 UI。
- **配置与模型层**：`Models/` 定义配置数据模型（如 `StartApplicationActionConfigModel`），实现 `ISerializableConfiguration` 接口进行 JSON 序列化；`ViewModels/` 配合 MVVM 模式处理复杂配置逻辑。
- **UI 表现层**：`GUI/` 和 `Views/` 存放基于 Windows Forms 的配置控件，采用 Code-Behind 模式（`.cs`, `.Designer.cs`, `.resx`）。
- **服务与工具层**：`Services/` 封装核心业务逻辑（如 `ApplicationLauncher`），`Utils/` 提供底层 API 调用（如 `WindowsShortcut`, `WindowActivator`）。
- **多语言支持**：`Language/` 模块通过 `PluginLanguageManager` 从嵌入的 XML 资源动态加载本地化字符串至 `PluginStrings` 单例。
