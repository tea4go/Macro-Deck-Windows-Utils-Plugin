- 动作类统一继承自 `PluginAction`，并通过重写 `Trigger` 方法执行核心逻辑，重写 `GetActionConfigControl` 返回对应的配置界面控件。
- 配置数据普遍采用 JSON 格式存储于 `Configuration` 字符串字段中，在 `Trigger` 或配置加载时使用 `JObject.Parse` 进行解析。
- 复杂配置场景采用 MVVM 模式，通过 `ISerializableConfigViewModel` 接口统一视图模型的行为，实现配置数据的序列化与保存。
- 本地化字符串通过 `PluginLanguageManager.PluginStrings` 静态属性访问，确保 UI 文本与业务逻辑解耦。
